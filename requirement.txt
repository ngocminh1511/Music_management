1. Giới thiệu đề tài

Website âm nhạc là hệ thống cho phép người dùng nghe nhạc trực tuyến, quản lý playlist, xem thông tin ca sĩ – bài hát, và quản trị kho dữ liệu âm nhạc. Đề tài sử dụng JSP/Servlet để xây dựng backend theo mô hình MVC, kết hợp MySQL làm cơ sở dữ liệu.

2. Lý do chọn đề tài

Nhu cầu nghe nhạc online cao, mô phỏng các nền tảng như Zing MP3, Nhaccuatui.

JSP/Servlet vẫn là nền tảng phổ biến trong đào tạo web Java.

Đề tài giúp hiểu sâu về:

MVC (phân tách Model – View – Controller)

Luồng xử lý Request/Response

Quản lý session, upload file (ảnh, audio)

Tương tác cơ sở dữ liệu với MySQL

3. Mục tiêu của đề tài
Mục tiêu tổng quát

Xây dựng một website nghe nhạc cơ bản, cho phép:

Người dùng xem, tìm kiếm, nghe nhạc.

Đăng ký / đăng nhập.

Tạo playlist cá nhân.

Quản trị viên quản lý kho nhạc, ca sĩ, thể loại, bài hát.

Mục tiêu cụ thể

Áp dụng mô hình MVC:

Model: Lớp xử lý dữ liệu, kết nối MySQL.

View: JSP hiển thị giao diện.

Controller: Servlet điều hướng và xử lý logic.

Xây dựng cơ sở dữ liệu chuẩn hoá, tối ưu truy vấn.

Tự triển khai player audio trên web (thẻ <audio>).

Xây dựng chức năng login với session, mã hoá mật khẩu (BCrypt).

Xử lý upload file nhạc + ảnh (Servlet 3.0 File Upload).

4. Phân tích yêu cầu
4.1 Chức năng người dùng
(1) Không cần đăng nhập

Nghe thử nhạc

Tìm kiếm (bài hát, ca sĩ, album)

(2) Có tài khoản

Nghe toàn bộ bài hát

Quản lý thông tin cá nhân

4.2 Chức năng quản trị (Admin)

Quản lý bài hát

Thêm file nhạc, ảnh bìa

Sửa thông tin

Xoá bài hát

Quản lý ca sĩ, thể loại

Quản lý tài khoản

Thống kê lượt nghe (nếu mở rộng)

Manage Users

6. Phân tích cơ sở dữ liệu (MySQL)
Các bảng chính đề xuất
users (id, username, password, email, role)
songs (id, title, singer_id, category_id, file_path, thumbnail, view_count, upload_date)
singers (id, name, description, avatar)
categories (id, name)
playlists (id, user_id, name)
playlist_song (playlist_id, song_id)

Đặc điểm

Chuẩn hoá 3NF

Sử dụng khóa ngoại (FK)

Tăng tốc tìm kiếm bằng index (title, singer_id)

7. Kiến trúc hệ thống (MVC)

Lớp kết nối DB (DBContext)

DAO: SongDAO, UserDAO, PlaylistDAO …

8. Quy trình xử lý ví dụ (Play Music)

User click bài hát → gửi request:
GET /song?id=10

Servlet nhận request → gọi SongDAO.getById(10)

Trả về Song object

JSP render giao diện + player nhạc

Nếu user nghe → tăng view_count
